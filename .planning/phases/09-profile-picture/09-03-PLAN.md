  <name>Task 1: Update DataStore and Repository</name>

    - app/src/main/java/com/jna/tictactoe/data/PreferenceRepository.kt

  <action>
    1.  In `UserPreferences.kt`, add `profilePicturePath: String?` to the data class.
    2.  In `PreferenceRepository.kt`, add a new `stringPreferencesKey` for the profile picture path.
    3.  Add a `suspend fun updateProfilePicturePath(path: String)` function to update the value in DataStore.
    4.  Update the `userPreferencesFlow` to read and expose the new `profilePicturePath`.

<task type="auto">
  <name>Task 2: Implement Image Saving in ViewModel</name>
  <files>
    - app/src/main/java/com/jna/tictactoe/screen/profile/ProfileViewModel.kt
  </files>
  <action>
    1.  Inject `Application` context into the `ProfileViewModel`.
    2.  Create a private suspend function `saveImageToInternalStorage(uri: Uri): String?`. This function should:
        a. Use the `ContentResolver` to get an `InputStream` from the content URI.
        b. Create a destination file in the app's internal storage (e.g., in `context.filesDir`).
        c. Copy the `InputStream` to the destination file.
        d. Return the absolute path of the newly created file.
    3.  Modify the `onImageSelected(uri: Uri?)` function:
        a. If the URI is not null, launch a coroutine.
        b. Call `saveImageToInternalStorage` with the URI.
        c. If saving is successful, call the repository's `updateProfilePicturePath` with the new file path.
  </action>
  <verify>
    Calling `onImageSelected` with a valid URI results in a file being created in the app's internal storage and its path being saved to DataStore.
  </verify>
  <done>
    ViewModel now handles saving the selected image and persisting its path.
  </done>
</task>
