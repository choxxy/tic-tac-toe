</objective>

<tasks>
<task type="auto">
  <name>Task 1: Display Profile Picture in ProfileScreen</name>
  <files>
    - app/src/main/java/com/jna/tictactoe/screen/profile/ProfileScreen.kt
  </files>
  <action>
    1.  In the `ProfileScreen` composable, collect the user preferences state from the `ProfileViewModel`.
    2.  Use the `AsyncImage` composable from Coil to display the avatar.
    3.  Set the `model` of `AsyncImage` to the `profilePicturePath` from the user preferences.
    4.  Provide a fallback drawable (e.g., a generic avatar icon) for the `error` and `placeholder` states of the `AsyncImage`. The path will be a local file path, so ensure Coil is correctly configured to handle it (it should be by default).
  </action>
  <verify>
    The avatar in `ProfileScreen` loads the image from the path provided by the ViewModel, or shows a placeholder if the path is null.
  </verify>
  <done>
    `ProfileScreen` displays the user's selected profile picture.
  </done>
</task>
<task type="auto">
  <name>Task 2: Display Profile Picture in MainMenuScreen</name>
  <files>
    - app/src/main/java/com/jna/tictactoe/screen/menu/MainMenuScreen.kt
  </files>
  <action>
    1.  The `MainMenuScreen` likely needs access to the user preferences. If it doesn't have a ViewModel, you may need to pass the `UserPreferences` object down from a higher-level composable or inject the `PreferenceRepository` if Hilt is setup. Given the project structure, it is likely already observing this state.
    2.  Locate the avatar display in `MainMenuScreen`.
    3.  Replace the static avatar with Coil's `AsyncImage`, similar to the implementation in `ProfileScreen`.
    4.  Use the `profilePicturePath` from the collected user preferences state as the `model`.
    5.  Provide the same fallback drawable as a placeholder.
  </action>
  <verify>
    The avatar in `MainMenuScreen` loads the image from the path provided by the user preferences state.
  </verify>
  <done>
    `MainMenuScreen` displays the user's selected profile picture.
  </done>
</task>

  <name>Checkpoint: Verify Profile Picture Flow</name>
  <what-built>
    The complete feature for selecting and displaying a profile picture.
  </what-built>
  <how-to-verify>
    1.  Launch the app and navigate to the **Profile Screen**.
    2.  Tap the edit icon on the default avatar. A dialog to choose 'Camera' or 'Gallery' should appear.
    3.  **Test Gallery:** Select 'Gallery', choose an image. The avatar on `ProfileScreen` should update to the selected image.
    4.  Navigate back to the **Main Menu**. The avatar there should also be updated.
    5.  **Test Camera:** Go back to the **Profile Screen**. Tap the edit icon, select 'Camera'. The camera app should open. Take a picture. The avatar on `ProfileScreen` should update to the new picture.
    6.  **Test Persistence:** Close the app completely and reopen it.
    7.  Check the **Main Menu** and **Profile Screen**. The avatar should be the last one you set.
  </how-to-verify>
