    - Guest taps a cell -> Guest sends `GameMessage.Move` to Host. Local state does NOT update yet.
    - Host receives `GameMessage.Move` -> Host validates move -> Host updates local `GameState` -> Host sends `GameMessage.SyncState` to Guest.

  </behavior>
  <action>

      - If player is GUEST, call `socketManager.send(GameMessage.Move(index))`.
      - If player is HOST, call `gameEngine.makeMove`, update local state, and call `socketManager.send(GameMessage.SyncState(newState))`.
  </action>

  </verify>
